# FreqTrade + 配置管理器 Dokploy 部署指南

## 📋 项目概述

本项目包含两个完全独立的服务：

1. **FreqTrade 核心服务** - 官方开源量化交易机器人
2. **配置管理 Web 服务** - 可视化配置界面，支持配置热重载

### 架构特点

- ✅ **完全独立**：配置服务可单独运行，不影响 FreqTrade
- ✅ **热重载**：配置修改后立即生效，无需重启 FreqTrade
- ✅ **新手友好**：可视化界面，详细的引导和提示
- ✅ **安全可靠**：默认模拟模式，自动备份配置

1. 点击 "Create Project"
2. 项目名称：`freqtrade`
3. 点击 "Create"

### 第三步：部署 FreqTrade 核心服务

#### 3.1 创建应用

1. 在项目内点击 "Create Application"
2. 选择 "Dockerfile" 部署方式

#### 3.2 配置构建设置

- **Build Path**: `./` (根目录)
- **Dockerfile**: `Dockerfile`
- **Port**: `8080`

无需额外环境变量

#### 3.4 配置存储卷

添加卷：
- **Path**: `/freqtrade/user_data`
- **Name**: `freqtrade_data`

#### 3.5 部署

点击 "Deploy" 按钮

---

### 第四步：部署配置管理服务

#### 4.1 创建应用

1. 点击 "Create Application"
2. 选择 "Dockerfile" 部署方式

#### 4.2 配置构建设置

- **Build Path**: `./config-server`
- **Dockerfile**: `Dockerfile`

- **Path**: `/freqtrade/user_data`

## 🖥️ 使用配置管理界面

### 访问地址

```
http://128.1.40.110:8000/balance
```

### 首次使用流程

#### 1️⃣ 检查状态

页面顶部显示三个状态：
- **配置服务**：应显示 ✅ 运行中
- **FreqTrade**：首次部署可能显示 ⏸️ 未运行（正常）
- **配置文件**：首次显示 ⚠️ 未创建（正常）

#### 2️⃣ 配置基本设置

点击 "⚙️ 基本设置" 标签：

**重要：保持模拟交易模式开启**
- ✅ 模拟交易模式（默认开启）
- 每笔交易金额：30 USDT（可调整）
- 最大同时持仓数：3（可调整）
- K线周期：5分钟（推荐）

#### 3️⃣ 配置交易所

点击 "🏦 交易所" 标签：

- 选择交易所：Binance（推荐）
- API Key：模拟模式可不填，实盘模式必填
- API Secret：模拟模式可不填，实盘模式必填

> 💡 **如何获取 API Key？**
> 1. 登录交易所官网
> 2. 进入账户设置 → API 管理
> 3. 创建 API，复制 Key 和 Secret

#### 4️⃣ 设置交易对

点击 "💱 交易对" 标签：

默认已添加：
- BTC/USDT
- ETH/USDT

可添加更多交易对，格式：`币种/USDT`

#### 5️⃣ 保存并生效

点击顶部 **"⚡ 保存并立即生效"** 按钮

## ⚡ 配置热重载说明

### 什么是热重载？

热重载是指**修改配置后无需重启 FreqTrade 服务，配置立即生效**。

### 热重载的工作流程

1. 你在网页上修改配置
2. 点击 "保存并立即生效"
3. 配置保存到文件
4. 自动调用 FreqTrade 的 API 重新加载配置
5. FreqTrade 继续运行，使用新配置

### 热重载的优势

- ✅ 不中断交易
- ✅ 立即生效
- ✅ 无需等待重启

### 如果热重载失败？

即使热重载失败，配置也已经保存成功：

- **情况1**：FreqTrade 未运行
  - 配置已保存，启动 FreqTrade 后会自动加载新配置

- **情况2**：API 认证失败
  - 检查用户名密码是否正确
  - 或手动重启 FreqTrade

- **情况3**：网络问题
  - 稍后重试，或手动重启

## 🔐 安全建议

在 "🔧 高级设置" → "Web UI 设置" 中：
- 修改默认用户名和密码

- 不要在公共场所展示 API Key
- 定期更换 API Key
- 使用 IP 白名单限制访问

### 3. 模拟模式测试

- 首次使用务必保持模拟模式
- 测试 1-2 周确认策略有效后再切换到实盘

## 🛠️ 故障排查

### 问题1：配置服务无法访问

**现象**：打不开 `http://128.1.40.110:8000`

**解决**：
1. 检查 Dokploy 中配置服务是否显示 "Running"
2. 检查端口是否为 8000
3. 检查防火墙设置

### 问题2：FreqTrade 未运行

**现象**：状态栏显示 "⏸️ 未运行"

**解决**：
1. 这是正常的，首次部署需要手动启动
2. 在 Dokploy 中找到 FreqTrade 应用
3. 点击 "Start" 或 "Deploy"

### 问题3：热重载失败

**现象**：提示 "热重载失败"

**解决**：
1. 检查 FreqTrade 是否已启动
2. 检查用户名密码是否正确
3. 手动重启 FreqTrade（配置仍然有效）

### 问题4：配置未生效

**现象**：保存后 FreqTrade 行为未改变

**解决**：
1. 检查是否点击了 "⚡ 保存并立即生效"
2. 检查热重载是否成功
3. 如失败，手动重启 FreqTrade

### 常见问题

**Q: 配置服务会影响 FreqTrade 运行吗？**
A: 不会。配置服务完全独立，即使配置服务停止，FreqTrade 也会继续正常运行。

**Q: 可以同时运行多个 FreqTrade 实例吗？**
A: 可以，但需要为每个实例分配不同的端口和数据卷。

**Q: 如何备份配置？**
A: 每次保存配置时系统会自动创建备份，可在 "📦 备份管理" 中查看和恢复。

**Q: 实盘模式安全吗？**
A: 实盘模式会使用真实资金交易，请确保：
1. 策略经过充分测试
2. 设置合理的止损
3. 从小资金开始
